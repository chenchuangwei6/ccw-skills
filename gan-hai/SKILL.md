# 赶海潮汐查询

## 数据文件

| 文件 | 内容 |
|------|------|
| `near-areas.json` | 12 个沿海区域，仅保留 ID + AreaName |
| `tide-stations.json` | 326 个潮汐站点，仅保留 Code + Name + AreaID |

## 脚本

| 脚本 | 用途 |
|------|------|
| `scripts/lookup_station.js` | 按区域名或站点名查找站点 |
| `scripts/query_tide.js` | 调用 API 查询指定站点日期的潮汐数据 |

---

## 查询流程

### 步骤 1：定位站点

根据用户输入选择匹配方式：

**用户提到区域名（如「广东」「山东」「浙江」）：**

```bash
node scripts/lookup_station.js <区域名>
```

输出 `type: "area"` 时，列出该区域下所有站点供用户选择。站点较多时（>10个），只展示前 10 个并提示用户缩小范围。

**用户提到站点名（如「庄河」「北戴河」）：**

```bash
node scripts/lookup_station.js <站点名>
```

输出 `type: "station"` 时，匹配到 1 个站点直接用；多个站点则列出让用户选择。

**用户提到非站点地名（如「渔人码头」「亚龙湾」）：**

脚本匹配不到任何站点（`type: "none"`）时，**不要直接说找不到**。利用你自身的知识完成以下推理链：

1. 判断该地名属于哪个沿海区域（如渔人码头→上海）
2. 用区域名再次执行 `node scripts/lookup_station.js <区域名>` 获取该区域所有站点
3. 基于你对地理位置的了解，从站点列表中选出**距离最近**的那个
4. 明确告知用户并继续查询：

> 「{地名}」不是潮汐监测站，距离最近的是 **{站点名}站**，以下潮汐数据可直接参考。

5. 如果无法判断该地名属于哪个区域或附近没有站点，列出全部 12 个区域供用户自行选择。

**用户未指定任何地点：**

```bash
node scripts/lookup_station.js
```

列出全部 12 个区域供用户选择，再进入区域→站点流程。

### 步骤 2：查询潮汐

确定站点 Code 后：

```bash
node scripts/query_tide.js <Code> <日期>
```

- 日期格式 `YYYY-MM-DD`，用户未指定则默认今天
- **注意：API 限制未登录用户只能查询近 3 天数据**，超出范围会返回错误

### 步骤 3：天气提示

**有天气/搜索类工具（如 weather skill、web search）：**

在查询潮汐后，主动获取站点的当日天气（气温、降雨、风力、浪高等），融入最终输出。

**没有任何天气获取工具：**

在输出末尾附上友好提示：

> ⚠️赶海前请关注当日天气——降雨、风力、气温都会影响安全与体验。建议确认天气后再出行。

### 步骤 4：格式化输出

脚本返回完整 JSON，包含以下字段：

```
station.name        — 站点名称
station.coordinate  — 坐标
station.benchmark   — 潮高基准面
tideEvents[]        — 高低潮事件（time, height, type）
ganhaiWindows[]     — 赶海建议窗口（低潮前后1小时）
hourlyTable[]       — 24小时逐时潮高
trends[]            — 涨落潮阶段描述（period, timeRange, direction, rangeDesc, ganhaiWindow）
summary             — 当日潮汐一句话摘要（最佳赶海时机、最大潮差）
```

## 特殊情况处理

| 情况 | 处理 |
|------|------|
| API 返回「未登录用户只允许查看近3天数据」 | 告知用户只能查近 3 天，引导用户指定近 3 天内的日期 |
| API 请求超时或网络错误 | 提示用户稍后重试，检查网络 |
| 站点匹配到 0 个结果 | 列出全部 12 个区域让用户选择 |
| 站点匹配到多个结果（同名站点） | 列出所有匹配项，标明所属区域，让用户选择 |
| 查询日期超出 API 数据范围 | 告知用户该日期暂无潮汐预报数据 |
